# Aegis Review collaboration instructions

- Read `docs/API.md`, `docs/SYSTEM_DESIGN.md`, your file in `docs/assignments/`, and your file in `prompts/` before editing.
- Work only on the fixed role branch and with the real member Git identity.
- Respect path ownership; stop and notify the leader when a contract change crosses roles.
- Use test-first development for behavior changes and run the role-specific checks before committing.
- Do not fabricate detections, task progress, Bug records, model metrics, screenshots, contributors, or Git authors.
- Do not commit runtime outputs, caches, secrets, `last.pt`, or complete training run directories.
- Merge through ordinary merge commits. Do not squash or rebase member contributions.
