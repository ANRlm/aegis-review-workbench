## Bug 处理流程

```text
QA 编写失败测试 + 证据
  → 通知责任成员
  → 责任成员在自己的分支和身份下修复
  → 修复 PR 合入 main
  → QA 用同一测试 + 自动测试复验
  → 在本文档填写修复提交与回归结果
```

## 记录模板

每个 Bug 包含：

- **Bug 编号**：BUG-XX
- **现象**：用户/测试者观察到的具体异常
- **环境**：Docker/host, Python 版本, 模型/分支
- **输入素材**：触发 Bug 的具体上传文件
- **复现步骤**：精确步骤（可被任何人执行）
- **失败命令或页面证据**：curl 输出 / 截图 / pytest 失败日志
- **根本原因**：代码层面的根因分析
- **责任模块**：后端 / CV / 前端
- **修复提交**：真实 SHA + 消息 + 修复人
- **回归命令**：确认修复的命令
- **回归结果**：通过 / 失败
