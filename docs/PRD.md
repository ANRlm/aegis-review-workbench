# 影鉴 Aegis Review 产品需求文档

## 1. 产品定位

影鉴面向游戏宣传素材、游戏录屏和数字媒体内容团队，提供“机器初筛 + 人工复核 + 结果留档”的桌面网页工作台。它解决长视频人工检查耗时、审核标准不统一、证据难追踪和交付资料分散的问题。

主要用户是素材审核员；项目管理员负责规则配置和任务清理；交付人员负责导出报告和验收包。

## 2. 成功标准

用户能够在一次连续操作中：

1. 上传一张合法图片或一个合法视频；
2. 获得任务编号，而不是等待完整推理；
3. 看到任务从 `created` 进入 `queued/running`，最终完成或失败；
4. 查看原素材、检测类别、置信度、边界框和至少一张证据帧；
5. 得到通过、待复核或不通过结论；
6. 填写负责人并人工改判；
7. 刷新页面后从历史任务重新打开报告；
8. 下载 JSON、CSV 和 ZIP 审核包。

验收时至少准备一个真实图片任务和一个真实视频任务，并能演示三种审核结论。

## 3. 功能范围

### 必须实现

- 单个 `jpg/jpeg/png/mp4/mov` 素材上传，最大 200MB；
- 自训练五类游戏 YOLO 模型推理；
- 视频每秒采样一次，最多 120 帧；
- 检测类别、置信度、`xyxy` 边界框记录；
- 三档可解释审核规则；
- 证据帧保存与展示；
- 人工改判、负责人和备注；
- 历史任务、统计数量与安全删除；
- JSON、CSV 和 ZIP 导出；
- 空、加载、失败、完成四种前端状态；
- 不支持格式、空文件、损坏媒体、模型缺失和任务异常提示。

### 平衡增强

- 本次任务规则和阈值可在页面修改；
- 通过、待复核、不通过统计；
- 审核负责人和备注；
- CSV 和 ZIP 审核包。

### 不在本次范围

- 用户登录和权限系统；
- 数据库、Redis、Celery 或云对象存储；
- 批量图片任务；
- 审核版本历史；
- 在线训练和服务启动时训练；
- 方向 B 的视频粗剪、片段排序和多规格输出。

## 4. 默认审核规则

```json
{
  "risk_classes": ["enemy"],
  "reject_confidence": 0.60,
  "review_confidence": 0.35,
  "inference_confidence": 0.25,
  "min_evidence_frames": 1,
  "sample_interval_seconds": 1.0,
  "max_sample_frames": 120
}
```

规则按顺序短路：

1. 任一 `enemy` 置信度不低于 `0.60`：不通过；
2. 最高风险置信度处于 `[0.35, 0.60)`：待复核；
3. 视频中低置信度风险帧比例处于 `(0.20, 0.80)`：待复核；
4. 处理正常且不满足前述条件：通过。

即使没有检测目标，也保存第一个代表帧并注明“未发现规则目标”。

## 5. 体验要求

最终页面使用暖白画布、石墨文字和低饱和青绿主色，橙色和红色只表达待复核与不通过。1366×768 首屏应能看到任务历史、素材/证据和审核面板。

上传成功后立即显示任务卡。轮询期间显示当前状态但不伪造进度百分比；失败时显示 `job.json` 中的真实错误。动画仅使用 160–240ms 淡入、状态切换和轻微位移，并支持 `prefers-reduced-motion`。

## 6. 验收底线

- README 命令能够启动；
- `/api/health` 正常并说明模型、FFmpeg、存储状态；
- 上传后先返回任务 ID；
- 状态真实变化；
- 至少一次真实自训练模型推理；
- 失败任务仍保留 `job.json`；
- 报告可以重新读取；
- 人工改判写回报告；
- 5 条正常、5 条异常测试和 2 个真实 Bug；
- 五名成员能通过提交记录说明自己的实际产出。
