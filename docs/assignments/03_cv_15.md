## 工作包

### C1：数据迁移与校验

- 迁移现有 96 张训练、24 张验证图片及对应标签；
- 保持类别顺序 `player/enemy/energy_orb/treasure_chest/health_potion`；
- `data.yaml` 使用项目内相对路径，不能写 `/workspace` 绝对路径；
- 校验图像与标签一一对应、类别 ID 0–4、归一化坐标位于 `[0,1]`；
- 在 `docs/MODEL.md` 记录数据来自 Day05/Day06 原创程序化游戏场景，不伪装为新采集真实照片。

### C2：可复现训练

- 使用 `yolo11n.pt` 初始化；
- 默认 30 epoch、640 图像、batch 2、CPU、固定 seed `20260718`；
- 训练输出写 `training_runs/day08_game_audit`；
- 只提交 `models/aegis_game_best.pt`、必要指标、曲线和 SHA-256；
- 不提交 `last.pt`、缓存、训练批次大图或整个 runs 目录；
- 训练不属于服务启动流程。

### C3：采样与检测

- 图片可解码为一个 frame；
- 视频读取 FPS、总帧数和时长；
- 以秒为单位采样并处理 FPS 缺失；
- 最多返回 120 帧；
- 失败时抛出可读领域异常；
- YOLO 适配器缓存单个模型实例；
- 框坐标裁剪到画面范围并转 Python `float`。

### C4：审核规则

用纯函数接收检测列表、采样帧数和 `AuditSettings`：

1. `max_enemy_conf >= reject_confidence` → `reject`；
2. `review_confidence <= max_enemy_conf < reject_confidence` → `review`；
3. 低置信度风险帧比例严格处于 `(0.20,0.80)` → `review`；
4. 否则 `pass`。

明确测试 `0.35`、`0.60`、`0.20`、`0.80` 边界。

### C5：证据与报告

- 按规则相关度、置信度和时间选证据；
- 至少保存一张 JPEG；
- 画框包含类别和两位小数置信度；
- 无目标时保存第一帧；
- 写 `analysis_report.json`、`detections.csv`；
- ZIP 包含 `job.json`、报告、CSV 和证据，不包含模型、数据集或原素材副本。

## 验收标准

- 模型真实存在且健康检查为 ready；
- 训练入口、参数、数据和指标可解释；
- 图片与视频均产生可序列化检测；
- 三档规则边界测试通过；
- 无检测时仍有证据；
- 模型缺失或视频损坏产生可读错误，而不是空成功报告。
